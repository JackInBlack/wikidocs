# Relazione Server SMTP
Installazione e configurazione di un Mail Server SMTP su macchina Unix.
Sistema Operativo: Raspbian GNU/Linux 10 (buster).
Versione SMTP: Postfix 3.4.14
Versione IMAP: Dovecot 2.3.4.1
Versione Antivirus: ClamAV 0.103.2

### Introduzione

Un servizio di posta elettronica fornisce il mezzo per inviare posta
elettronica tra nodi all’interno di una rete di calcolatori. Opera tramite
il protocollo **Simple Mail Transfer Protocol** (SMTP) il quale si appoggia
sulla suite TCP/IP. Si tratta di un modello di comunicazione client-
server. Il client apre un canale di comunicazione TCP sulla porta 25
(porta 587 per SSMTP Secure SMTP) con il server e comunica ad esso il
destinatario della email. Una volta che il server accetta la destinazione,
il client può inviare il contenuto della email. Il mail server a cui si
connette inizialmente il client non è sempre il nodo di rete finale della
corsa dell’email difatti il protocollo SMTP è progettato per inoltrare la
posta elettronica ad altri mail server fino a quando non viene raggiunto
il nodo destinatario. I server intermediari sono chiamati MTA(Mail
Transfer Agent). Una volta che l’email è giunta al nodo destinazione il
server finale salva nel proprio File System l’intera email.
Il formato dati di un’email è dettato dagli RFC 5322, 2045 e 2049. Le
componenti principali di un’email sono header e body.
**HEADER**
L'intestazione è composta da una serie di righe, ognuna composta da un
tipo: valore, tra i quali:
From: indirizzo mittente
To: mailbox destinatario (anche più destinatari)
Date: data di spedizione
Subject: soggetto del messaggio
Cc: copia destinatari
Replay-To: indirizzo per la risposta
Message-Id: identificatore unico del messaggio
BODY
Il corpo del messaggio è il testo dell'email, in formato ASCII.
Un indirizzo di posta elettronica è formato da due campi, ovvero una parte
che rappresenta il nome utente del destinatario ed una parte che
rappresenta il dominio.
Ad esempio l’indirizzo di posta elettronica
pi@gruppo9.labreti.it
ha come nome utente “pi” e come dominio “gruppo9.labreti.it”.
